## 에이전트 개발 계획 수립

### 1. 요구사항 분석
```
단계:
1. 비즈니스 목표 정의
2. 사용자 스토리 작성
3. 기능 요구사항 도출
4. 비기능 요구사항 정의
5. 제약사항 파악
```

### 2. 아키텍처 설계
```
고려사항:
- 에이전트 구조 (단일 vs 다중 스킬)
- 외부 시스템 통합 방식
- 데이터 흐름 설계
- 에러 처리 전략
- 확장성 및 유지보수성
```

### 3. 스킬 분해
```
원칙:
- 단일 책임 원칙 적용
- 재사용 가능한 단위로 분리
- 의존성 최소화
- 명확한 인터페이스 정의
```

## 협업 가이드

### Git 워크플로우
```
main (프로덕션)
  ↑
develop (개발)
  ↑
feature/* (기능 개발)
```

### 커밋 메시지 규칙
```
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 포맷팅
refactor: 코드 리팩토링
test: 테스트 추가/수정
chore: 빌드 프로세스 수정
```
